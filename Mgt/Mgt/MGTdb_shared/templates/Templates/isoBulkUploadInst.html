How to fill in the spreadsheet to upload multiple isolates


Required fields (or columns)

The following fields are required:
  • Isolate identifier
  • Allele filename
  • Privacy status
  • Continent
  • Country

Additional fields (or columns)

  • State
  • Postcode
  • Source
  • Type
  • Host
  • Disease
  • Date
Field name Is it required Format Description
Isolate identifier Yes Any alphabets, numbers, _ or - . An identifier to the isolate. The chosen name has to be unique within a project.
Allele filename Yes Any alphabets, numbers, ., _ and - . A filename (with a .fasta or .fa extention). A file with this name will be expected in stage 3 of the bulk allele upload process.
Privacy status Yes Either PU or PV . Choose PU (for public) to make the Isolate and its meta data public and searchable by anyone using the database. Choose PV (for private) to make the Isolate and its meta data private and only searchable by the uploader.
Continent Yes Either of the following options: Africa , Antarctica , Oceania , North America , South America , Europe or Asia. This field represents the continent in which the isolate was isolated.
Country Yes The full list of allowable countries is presented below. This field represents the country in which the isolate was isolated.
State (or sub-country) No Text. This field represents the state (or sub-country) in which the isolate was isolated. For example, "NSW" in Australia, or "England" in UK.
Postcode (or Zipcode) No Numbers. This field represents the postcode (or zipcode) in which the isolate was isolated.
Source No Any text. Here, source represents, the sample source (or product). The source can be organic or non-organic. Some examples are: Eggs, Beef, Misc Food, live animal, Human.
To clarify, if for example, the "isolation host" is "chicken", the sample source may be "eggs". Another example, if the "isolation host" is "human", the sample source may also be "human".
Type No Any text. Type represents the setting in which the isolate was isolated. Some examples are: Clinical, Other.

The main goal of this field is to distinguish between human clinical samples and other samples (such as vetenary samples).
Host No Any test. Here, host represents the isolation host organism e.g. chicken, Human.
Disease No Any text. Disease refers to the disease that was observed in the isolated host. Some examples are: Gastroenteritis, sepsis.
Date No In the format: YYYY.MM.DD or YYYY-MM-DD This field represents the date on which this isolate was isolated e.g. 2020-11-03 or 2020.11.03
Year Yes Numbers. This field represents the year in which this isolate was isolated. The year provided here, and the year in the date are checked to make sure that they match.
Month No Numbers, ranging from 1 to 12. The month in which this isolate was isolated. The month provided here, and the month in the date are checked to make sure that they match. If month is not provided, and date is provided, this field is automatically extracted.
